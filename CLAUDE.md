## Project: rust-ir

Typed Rust IR (Intermediate Representation) builder for JSSG transforms. Part of the [refactory-lang](https://github.com/refactory-lang) organization. **Generated from ir-codegen** -- 134 node kinds from the tree-sitter-rust grammar. Builds structured Rust AST nodes from grammar-derived types, renders them to Rust source strings, and validates via tree-sitter parse. Zero runtime npm dependencies -- runs inside the Codemod JSSG runtime.

### Generation

This package is generated by `@refactory/ir-codegen`. To regenerate:

```bash
cd ../ir-codegen
npx tsx scripts/generate-rust-ir.ts ../rust-ir/src
```

**Generated files** (do not hand-edit): `types.ts`, `fluent.ts`, `nodes/*.ts`

**Hand-written files** (preserved across regeneration): `render.ts`, `validate-fast.ts`, `render-valid.ts`

### Architecture

The package follows a **Build -> Render -> Validate** pipeline:

1. **Build** -- Fluent builders (`ir.*` namespace) or factory functions (`src/nodes/*.ts`) create typed IR nodes from grammar-derived types
2. **Render** -- `render(node)` renders with validation (default), `renderSilent(node)` renders without validation
3. **Validate** -- `validate(source)` parses via tree-sitter, `validateFast(source)`/`assertValid(source)` use regex-based validation

- **Fluent builders** (`src/fluent.ts`): `ir.*` namespace implementing `BuilderTerminal` from `@refactory/grammar-types` -- the preferred API. 134 node kinds.
- **Node builders** (`src/nodes/`): 134 factory functions + fluent Builder classes, one per Rust grammar node kind (generated from ir-codegen)
- **Types** (`src/types.ts`): Grammar-derived TypeScript types for all 134 IR nodes (generated)
- **Render** (`src/render.ts`): Recursive renderer that converts IR nodes to Rust source (hand-written)
- **Render + Validate** (`src/render-valid.ts`): `render()` (with validation) and `renderSilent()` (without) pipeline (hand-written)
- **Validate** (`src/validate.ts`): Tree-sitter-based validation for rendered output
- **Validate Fast** (`src/validate-fast.ts`): Regex-based `validateFast()` and `assertValid()` for lightweight validation (hand-written)
- **Entry** (`src/index.ts`): Re-exports all public API
- **Tests** (`tests/`): Vitest test suite

### Fluent Namespace Keys

The `ir.*` namespace uses grammar-derived names, not abbreviated aliases:

| Fluent key | Grammar node |
|---|---|
| `ir.structItem(name)` | `struct_item` |
| `ir.fn(name)` | `function_item` |
| `ir.implItem(type)` | `impl_item` |
| `ir.useDeclaration(arg)` | `use_declaration` |
| `ir.macro_invocation(name)` | `macro_invocation` |
| `ir.attributeItem(value)` | `attribute_item` |
| `ir.ifExpression(cond)` | `if_expression` |
| `ir.file()` | `source_file` |

Builder methods use grammar field names: `.children([...])`, `.parameters(...)`, `.returnType(...)`, `.body(...)`.

### Conventions

- All node types are 100% grammar-derived from `@codemod.com/jssg-types`
- Generated files should not be hand-edited; regenerate from ir-codegen instead
- Hand-written files (`render.ts`, `validate-fast.ts`, `render-valid.ts`) are preserved across regeneration
- Package ships TypeScript source directly (`src/index.ts` as main entry); consumers must use a TS-aware bundler or the Codemod JSSG runtime
- Uses `pnpm` as package manager
- Type checking via `tsgo` (native TypeScript compiler)


### Speckit Workflow

This repo uses [speckit](https://github.com/speckit) for specification-driven development.

- **Specs**: `specs/<NNN-feature-name>/spec.md` — feature specifications
- **Plans**: `specs/<NNN-feature-name>/plan.md` — implementation plans with tasks
- **Checklists**: `specs/<NNN-feature-name>/checklists/` — quality gates
- **Templates**: `.specify/templates/` — spec, plan, task, checklist templates
- **Extensions**: `.specify/extensions/` — verify, sync, review, workflow hooks

**Branch convention**: Feature branches are named `<NNN>-<short-name>` matching the spec directory (e.g., `001-milestone1-pipeline`).

**Issue -> Spec flow**: Issues labeled `ready-to-spec` trigger the `ready-to-spec-notify` workflow, which assigns Copilot to run the speckit workflow and produce a spec + plan + tasks.
